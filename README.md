# СИСТЕМА УПРАВЛЕНИЯ ЗАКАЗАМИ И КОНТРОЛЕМ ПРОИЗВОДСТВА ДЛЯ КОМПАНИИ ПО ИЗГОТОВЛЕНИЮ ИНДИВИДУАЛЬНЫХ СУВЕНИРОВ

Ключевым аспектом проекта является создание единого информационного пространства, объединяющего процессы работы с клиентами, управления заказами и контроля производства. Система должна обеспечить сквозную координацию между коммерческой и производственной деятельностью через автоматизированный процесс передачи заказов от менеджеров к мастерам производственного участка.
Техническая архитектура решения предусматривает реализацию основных бизнес-операций, включая управление клиентской базой, создание и отслеживание заказов, назначение исполнителей, обновление статусов выполнения и формирование отчетности. 
Важным требованием является обеспечение безопасности данных и разграничения прав доступа между двумя категориями пользователей –менеджерами и мастерами производственного участка. Система должна предоставлять соответствующий набор функций для каждой роли, предотвращая несанкционированный доступ к конфиденциальной информации.

Клиентская часть системы реализуется с использованием базовых веб-технологий – HTML5, CSS3 и JavaScript. HTML5 отвечает за семантическую структуру интерфейса, CSS3 обеспечивает адаптивное визуальное оформление, а JavaScript добавляет интерактивность и динамическое поведение элементам системы. Для реализации асинхронного взаимодействия с сервером используется технология AJAX.
Серверная часть системы строится на платформе Node.js в сочетании с фреймворком Express.js.
Для хранения данных выбрана документоориентированная СУБД MongoDB, что полностью соответствует требованиям гибкости структуры хранения информации. 
Архитектурный подход к построению системы основан на комбинации паттерна MVC (Model-View-Controller) и REST API, что обеспечивает четкое разделение ответственности между компонентами приложения. Клиентская часть взаимодействует с сервером через REST API.

Для управления доступом к системе реализована двухэтапная процедура, сочетающая аутентификацию и авторизацию. Аутентификация пользователей выполняется с использованием токенов доступа формата JSON Web Token (JWT), генерируемых серверной частью приложения на платформе Node.js.
Конфиденциальность учётных данных обеспечивается криптографическим преобразованием паролей перед их сохранением в базу данных. Для этого применяется алгоритм хеширования с использованием библиотеки bcrypt, которая генерирует уникальные криптографические хеши для каждого пароля.
